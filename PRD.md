# PRD: Infra-Miru

## 概要
Kubernetesクラスタのリソース利用率を可視化し、過剰プロビジョニングを検出してコスト最適化を提案するダッシュボード。

## 背景・動機
- Kubernetesクラスタの多くでCPU/メモリのRequest値が実使用量を大幅に上回っている
- 過剰プロビジョニングは直接的なコスト増加に繋がるが、具体的な金額が見えにくい
- 日本のSaaS企業で多い「夜間・休日のスケールダウン忘れ」が慢性的なコスト浪費の原因

## ユニークポイント
Pod単位のRequest vs 実使用量の乖離を日本円ベースのコスト削減額に換算し、夜間・休日のスケールダウン忘れを自動検出してCronHPA設定を生成する。

## 技術スタック
- **フロントエンド**: TypeScript / Next.js（リソースダッシュボードUI）
- **バックエンド**: Go（API / Kubernetes API連携 / リソース分析 / コスト計算）
- **データストア**: PostgreSQL（リソース履歴・コスト・提案DB）
- **メトリクス**: Prometheus / Grafana（メトリクス収集・可視化）

## MVP機能と受け入れ条件

### F1: Kubernetesクラスタ接続とPod単位リソースダッシュボード
- [x] Go APIがkubeconfigを使用してKubernetesクラスタに接続できる
- [x] Pod単位のCPU/メモリ利用率を取得するAPIエンドポイントが存在する
- [x] Next.jsダッシュボードでPod一覧とリソース利用率を表示できる
- [x] Namespace/Deployment単位でフィルタリングできる

### F2: Request vs 実使用量の乖離分析とコスト削減提案
- [x] Pod単位のRequest値と実使用量の乖離率を計算するロジックが存在する
- [x] 乖離率に基づいて日本円ベースのコスト削減額を計算できる
- [x] 「このDeploymentはRequestの15%しか使っていない→Requestを50%削減で月¥X節約」形式の提案を生成できる
- [x] 提案一覧をダッシュボードで表示できる

### F3: 夜間・休日低負荷検出とCronHPA設定テンプレート生成
- [x] 時間帯別の負荷パターンを分析するロジックが存在する
- [x] 夜間（22:00-06:00）・休日の低負荷時間帯を自動検出できる
- [x] 検出結果に基づいてCronHPA設定テンプレートをYAML形式で生成できる
- [x] 生成されたテンプレートをダッシュボードからコピーできる

## スコープ外（本番利用前に必要）

- マルチクラスタ対応
- Slack/Teams通知連携
- カスタムコスト単価設定UI
- Prometheus自動セットアップ
